# Security Policy

## Reporting a Vulnerability

Please do not open public issues for unpatched vulnerabilities.

Report security issues privately to the maintainer with:

- Impact summary
- Reproduction steps
- Affected versions/commit if known
- Suggested fix (optional)

## Response Expectations

- Initial acknowledgment target: within 72 hours
- We may request extra details or a proof-of-concept
- If confirmed, a fix will be prepared and released through GitHub Releases

## Supported Versions

We generally support the latest released version.
You may be asked to reproduce issues on the latest release.

## Disclosure

After a fix is available, we may publish:

- Impact summary
- Affected/fixed versions
- Recommended upgrade path
