# Support

Thank you for using kra.
Please choose the right channel based on your purpose.

---

## Questions (usage / general help)

- GitHub Discussions:
  https://github.com/tasuku43/kra/discussions

Before asking, please check:

- `README.md`
- `docs/`

### Information to include

- The command you ran
- Expected result / actual result
- `kra version` (or `kra --version`)
- OS / shell (example: macOS 14 / zsh)
- Relevant logs (mask sensitive data)

---

## Bug reports

- GitHub Issues:
  https://github.com/tasuku43/kra/issues/new

### Information to include

- Short summary of the bug
- Reproduction steps (minimal reproduction preferred)
- Expected behavior / actual behavior
- `kra version` (or `kra --version`)
- OS / execution environment
- Logs or stack traces
- Relevant workspace/root state (as much as you can share)

---

## Feature requests

- GitHub Issues (feature request)
- Or GitHub Discussions:
  https://github.com/tasuku43/kra/discussions

### Information to include

- The problem you want to solve
- Expected UX (CLI examples / option ideas)
- Workarounds you use today
- Scope (MVP vs future extension)
- Compatibility impact (breaking or non-breaking)

---

## Security reports

Please do not post unpatched vulnerabilities in public issues.
See `SECURITY.md` for the private reporting process.

---

## Support scope

- Responses are best-effort.
- Deep environment-specific consulting is not guaranteed.
