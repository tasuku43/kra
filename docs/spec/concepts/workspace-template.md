# Workspace templates

## Purpose

Define how workspace templates are stored and validated before `ws create`.

## Location

- Templates are root-local:
  - `<current-root>/templates/<name>/`
- `<name>` validation follows workspace ID rules (`gion`-compatible; no path separators).

## Copy model

- `ws create` copies template content from `templates/<name>/` to `workspaces/<id>/`.
- Copy is static (no variable interpolation / placeholder expansion).
- `.gionx.meta.json` is not template-managed; it is always generated by system.

## Reserved paths (top-level only)

The following paths are forbidden at template root (`templates/<name>/`):

- `repos/`
- `.git/`
- `.gionx.meta.json`

If any reserved path exists, validation fails.

## Entry constraints

- Symlinks are forbidden.
- Unsupported special file types are forbidden.
- Validation reports all found violations in one run.

## Validation reuse

One shared validation implementation is used by:

- `gionx ws create` (preflight before any workspace directory creation)
- `gionx template validate`
